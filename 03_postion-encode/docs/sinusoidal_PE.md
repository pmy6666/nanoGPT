# 《Sinusoidal Position Encoding 底层原理》

## 1. 这份技术到底在解决什么问题

前一篇你已经弄清楚了：

- `token embedding` 负责表示“这个 token 是什么”
- Transformer 的 `self-attention` 本身不天然感知顺序

所以还缺一块信息：

**这个 token 在序列中的什么位置。**

`Attention Is All You Need` 给出的解决方案之一，就是 **Sinusoidal Position Encoding（正弦位置编码）**。

它的目标不是再学一个“词义向量”，而是给每个位置 `pos` 构造一个固定的、可计算的向量 `PE(pos)`，然后把它加到 token embedding 上：

```text
input representation = token embedding + positional encoding
```

这样模型一开始拿到的输入，就同时包含：

- 词的内容信息
- 词的位置信息

---

## 2. 论文中的原始公式

在 `Attention Is All You Need` 里，位置编码的定义是：

```text
PE(pos, 2i)   = sin(pos / 10000^(2i / d_model))
PE(pos, 2i+1) = cos(pos / 10000^(2i / d_model))
```

这里每个符号的意思要非常清楚：

- `pos`：序列中的位置，比如第 0、1、2、3 个 token
- `i`：位置编码向量中的“频率编号”
- `d_model`：模型隐藏维度，比如 128、256、768
- `2i`：偶数维，用 `sin`
- `2i+1`：奇数维，用 `cos`

换句话说：

- 第 0 维和第 1 维是一对
- 第 2 维和第 3 维是一对
- 第 4 维和第 5 维又是一对

每一对维度都在描述“位置”，只是**使用的频率不同**。

---

## 3. 先别急着背公式，先看它长什么样

如果 `d_model = 8`，那么某个位置 `pos` 的位置编码大致像这样：

```text
[
  sin(pos / 1),
  cos(pos / 1),
  sin(pos / 10),
  cos(pos / 10),
  sin(pos / 100),
  cos(pos / 100),
  sin(pos / 1000),
  cos(pos / 1000),
]
```

真实公式不是严格的 `1, 10, 100, 1000`，但直觉上可以这样理解：

- 前面的维度变化得很快，频率高
- 后面的维度变化得很慢，频率低

所以同一个位置 `pos`，会被投影成一组“快波 + 慢波”的组合信号。

这有点像：

- 不是只用一个时钟记录时间
- 而是同时用很多个不同速度的时钟记录时间

这样每个位置就更容易被唯一地区分出来。

---

## 4. 数学直觉一：为什么要用正弦和余弦

正弦和余弦有 3 个非常关键的性质。

### 4.1 它们是连续、平滑的函数

相邻位置的编码不会突然跳变。

比如 `pos=10` 和 `pos=11`，它们对应的值会比较接近；  
`pos=10` 和 `pos=1000`，差异会更大。

这很符合“位置”本身的直觉：

- 相邻位置应该更相似
- 远距离位置应该更不同

### 4.2 它们天然带周期性

一个正弦波可以表示一种节奏，但单一频率会重复，所以论文没有只用一个波，而是用了很多不同频率的波。

结果就是：

- 高频维度负责精细地区分近处位置
- 低频维度负责稳定表示远距离位置

多个频率叠加后，位置表示会更丰富。

### 4.3 它们对“位移”有很好的代数结构

这点最重要。

设：

```text
theta = pos / 10000^(2i / d_model)
delta = k   / 10000^(2i / d_model)
```

那么：

```text
sin(theta + delta) = sin(theta)cos(delta) + cos(theta)sin(delta)
cos(theta + delta) = cos(theta)cos(delta) - sin(theta)sin(delta)
```

这说明：

**位置 `pos + k` 的编码，可以由位置 `pos` 的编码通过一个只和偏移量 `k` 有关的线性变换得到。**

这就是论文里这套设计最核心的数学直觉之一：

模型更容易从绝对位置编码里，推导出相对位置信息。

---

## 5. 数学直觉二：为什么一半维度用 sin，一半维度用 cos

如果只用 `sin`，会丢掉一部分相位信息。

把 `sin` 和 `cos` 成对使用，本质上是在每个频率上构造一个二维向量：

```text
[sin(theta), cos(theta)]
```

你可以把它看成单位圆上的一个点。

当位置从 `pos` 变成 `pos + k` 时，这个点会在二维平面里旋转一个角度。

于是：

- 同一个频率上的位置变化，变成了一个“旋转”
- 不同频率上，又有不同速度的旋转

这就是为什么 sinusoidal encoding 很有几何感。

它不是随便拼了一堆数，而是在不同二维子空间里做不同速度的旋转表示。

---

## 6. 数学直觉三：为什么是 `10000^(2i / d_model)`

这个项决定了不同维度的频率尺度。

你可以把它理解成一个**指数级拉开的频率刻度表**。

如果写成角频率形式，更容易理解：

```text
omega_i = 1 / 10000^(2i / d_model)
```

那么公式就变成：

```text
PE(pos, 2i)   = sin(pos * omega_i)
PE(pos, 2i+1) = cos(pos * omega_i)
```

这表示：

- 小的 `i` 对应大的 `omega_i`，变化快
- 大的 `i` 对应小的 `omega_i`，变化慢

为什么要这样设计？

因为语言里既需要：

- 短距离分辨率
- 长距离位置趋势

如果所有维度频率都一样，那么位置表示会非常单一。  
如果频率跨度太窄，又难以同时覆盖近距离和远距离。

指数级分布正好能让不同维度关注不同尺度的位置变化。

---

## 7. 它为什么能帮助模型理解相对位置

这个问题非常关键，因为很多人只知道它能表示“绝对位置”，但不知道它为什么也有利于“相对位置”。

对某个固定频率来说，位置编码的一对维度是：

```text
[sin(pos * omega), cos(pos * omega)]
```

而 `pos + k` 的对应编码是：

```text
[sin((pos + k) * omega), cos((pos + k) * omega)]
```

根据三角恒等式，它总能写成：

```text
[sin((pos + k) * omega), cos((pos + k) * omega)] = A(k) [sin(pos * omega), cos(pos * omega)]
```

其中 `A(k)` 是一个只依赖 `k` 的 2x2 变换矩阵：

```text
A(k) = [
  [ cos(k * omega),  sin(k * omega)],
  [-sin(k * omega),  cos(k * omega)]
]
```

注意这里最关键的不是你把矩阵背下来，而是你要明白：

**从 `pos` 到 `pos+k` 的变化规律是稳定的，并且只由偏移量 `k` 决定。**

所以模型在学习过程中，更容易把：

- “前一个词”
- “后两个词”
- “距离 5 的词”

这类关系从编码结构里挖出来。

## 8. 它和 learned positional embedding 的本质区别

Sinusoidal Position Encoding 是：

- 固定的
- 不参与训练
- 只要知道位置和维度，就能直接算出来

而 learned positional embedding 是：

- 每个位置一个可学习向量
- 训练中更新参数

sinusoidal 的一个重要优点是：

**对训练时没见过的更长序列，理论上也可以继续外推计算位置编码。**

因为它不是查表，而是公式生成。

当然，能不能真正泛化到很长序列，还要看模型整体训练情况，但从编码本身来说，它是可外推的。

---

## 9. 为什么把它“加到” token embedding 上，而不是拼接

原论文的做法是直接相加：

```text
x_input = token_embedding + positional_encoding
```

这样做有几个现实原因：

### 9.1 维度不变

如果拼接，输入维度会变成两倍，后续所有线性层参数都得跟着改。

相加则不会改变 `d_model`。

### 9.2 每个位置都同时携带内容和位置

相加之后，同一个向量里既有“词义”也有“位置”。

后续线性层可以自己学习如何使用这两类信息。

### 9.3 工程上简单

实现成本低，效率也高。

---

## 10. 你应该真正记住的直觉

可以把 sinusoidal position encoding 理解成下面这句话：

**每个位置都被映射成一组不同频率的旋转坐标；近处位置和远处位置会呈现不同的相位关系，而这种相位关系有助于 attention 学习顺序和相对距离。**

如果再压缩成三句话，就是：

1. 它用不同频率的 `sin/cos` 给每个位置生成唯一向量。
2. 低维变化快，高维变化慢，因此同时覆盖短距离和长距离。
3. 位置偏移对应稳定的相位变化，所以模型更容易学习相对位置关系。

## 12. 最后一句总结

Sinusoidal Position Encoding 的本质，不是“给 token 编个序号”这么简单，而是：

**用多尺度的周期函数，把位置变成一种既能表示绝对位置、又对相对位移有良好结构的连续向量表示。**
