# Normalization 总结

## 1. 为什么需要 Normalization

Normalization 的核心目的，不是让数据“好看”，而是让不同层、不同样本、不同训练阶段的数值尺度尽量稳定。

如果每一层的激活值忽大忽小，模型训练会出现几个典型问题：

- 梯度不稳定，容易震荡；
- 不同层对学习率的敏感程度差异很大；
- 某些特征维会异常大，压制其他维度；
- 训练速度变慢，优化更难收敛。

因此，各种 Norm 方法本质上都在做同一件事：

“先把数值拉回一个稳定范围，再交给模型继续学习。”

它们的主要区别，不在于是否归一化，而在于：

- 沿哪个维度统计；
- 是否依赖 batch；
- 是否减去均值；
- 更适合哪类网络结构。

## 2. 三种 Norm 的核心差别

### BatchNorm

BatchNorm 的核心思想是：

对同一特征维，在一个 mini-batch 的不同样本之间做标准化。

它关注的是：

“某个样本在这一维上，相对这个 batch 的平均水平偏高还是偏低。”

所以 BatchNorm 强依赖 batch 统计量。一个 batch 中样本分布如果比较稳定，那么它的效果通常很好。

### LayerNorm

LayerNorm 的核心思想是：

对单个样本内部的特征维做标准化。

它关注的是：

“在这个样本内部，某一维相对其他维度是大还是小。”

它不依赖 batch，完全由样本自身决定归一化结果，因此对 batch size 不敏感。

### RMSNorm

RMSNorm 的核心思想是：

不做减均值，只用均方根控制向量整体尺度。

它关注的是：

“这个向量整体幅值是不是太大，而不是它的均值是不是偏移了。”

所以它比 LayerNorm 更简洁，也保留了输入中的均值信息。

## 3. 从数学直觉看三者区别

可以把三种方法理解为三种不同的“参照系”。

### BatchNorm 的参照系是整个 batch

它会把一个样本放到同一批样本里去比较。

这意味着：

- 如果 batch 内样本分布稳定，归一化效果好；
- 如果 batch 内样本差异很大，统计量会抖动；
- batch size 太小时，均值和方差估计不稳定。

### LayerNorm 的参照系是样本自己

它不需要看别的样本，只看当前向量内部各维度的相对关系。

这意味着：

- batch size 为 1 也能正常工作；
- 序列长度变化不会直接破坏归一化逻辑；
- 更适合 token-level、sample-level 的建模。

### RMSNorm 的参照系是向量能量

它甚至不关心中心位置，只关心整体长度或能量是不是过大。

这意味着：

- 它做的是更弱、更轻量的约束；
- 数值更稳定，但干预比 LayerNorm 更少；
- 在大模型中常常能以更简单的形式获得接近甚至更好的效果。

## 4. 适用性总结

### BatchNorm 更适合 CNN

BatchNorm 最典型的成功场景是 CNN，原因有几个：

- 图像任务中，一个 batch 内样本分布往往相对稳定；
- CNN 的特征图具有较强局部结构，相邻样本统计特征比较接近；
- 视觉任务通常可以使用较大的 batch size，这使得 batch 统计更可靠；
- BatchNorm 在卷积网络中已经被大量实验验证，效果稳定。

因此在传统 CNN 中，BatchNorm 往往是默认选择之一。

### BatchNorm 通常不适合 LLM

在 LLM 或更一般的 Transformer 中，BatchNorm 通常不是主流选择，核心原因是：

- 序列长度常常变化，样本之间差异很大；
- 不同 batch 的 token 分布波动明显，统计量不稳定；
- 语言建模经常受显存限制，batch size 可能较小；
- 自回归建模更强调每个 token、每个样本自身的稳定性，而不是跨样本对齐。

可以直观理解为：

图像 batch 往往像“一组相似分布的数据”，而语言 batch 更像“很多长度、主题、结构都不同的序列拼在一起”。

这会使 BatchNorm 依赖的 batch 统计量变得不够稳定，进而影响训练一致性。

## 5. 为什么 LayerNorm 更适合 Transformer / LLM

LayerNorm 更适合 Transformer，主要因为它不依赖 batch。

这带来几个直接优势：

- 小 batch 也稳定；
- 每个 token 可以独立归一化；
- 训练和推理行为更一致；
- 更适合变长序列。

Transformer 的输入通常是 `[B, T, D]`，LayerNorm 沿最后一维 `D` 做归一化，这非常符合“每个 token 独立建模”的结构特点。

所以在大多数标准 Transformer 架构中，LayerNorm 是最常见的选择。

## 6. 为什么很多新 LLM 偏向 RMSNorm

RMSNorm 可以看成是 LayerNorm 的简化版。

它相比 LayerNorm 的优势主要在于：

- 计算更简单；
- 少了一步减均值；
- 对尺度进行约束，但不过度干预表示；
- 在大模型中实践效果很好。

很多现代 LLM 使用 RMSNorm，原因并不是“LayerNorm 不行”，而是：

在超大模型里，RMSNorm 常常能用更简洁的归一化方式获得足够稳定的训练，并带来更高的实现和计算效率。

## 7. 一个简洁的对比结论

可以把三者记成下面这组直觉：

- BatchNorm：我和这个 batch 里的别人比，适合 CNN。
- LayerNorm：我和我自己内部各维比，适合 Transformer。
- RMSNorm：我只控制自己的整体尺度，适合现代大模型中的轻量归一化。

## 8. 最终建议

如果你的任务是：

- CNN / 图像模型：优先考虑 BatchNorm；
- Transformer / NLP / LLM：优先考虑 LayerNorm 或 RMSNorm；
- 更强调经典稳定性和通用性：选 LayerNorm；
- 更强调简洁、高效、现代大模型实践：选 RMSNorm。

如果一句话总结：

BatchNorm 依赖“群体统计”，LayerNorm 依赖“个体内部统计”，RMSNorm 依赖“个体整体能量统计”。
