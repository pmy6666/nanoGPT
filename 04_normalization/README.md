# 04 Normalization

本目录主要整理了三种常见归一化方法的最小实现与配套说明：

- `BatchNorm`
- `LayerNorm`
- `RMSNorm`

这一部分的重点不是工业级封装，而是从数学公式、直觉理解和最小代码实现三个层面，把归一化的核心思想讲清楚。

## 目录结构

- `Batchnorm.py`
  `BatchNorm` 的最小实现，面向二维输入 `[B, C]`，适合教学和理解 batch 维度上的归一化。
- `Layernorm.py`
  `LayerNorm` 的最小实现，沿最后一维做标准化，适合 `[B, D]` 或 `[B, T, D]` 形式输入。
- `RMSnorm.py`
  `RMSNorm` 的最小实现，沿最后一维做均方根归一化，不减均值。
- `docs/BatchNorm.md`
  介绍 BatchNorm 的公式、数学直觉、实现解释和简单数值例子。
- `docs/LayerNorm.md`
  介绍 LayerNorm 的公式、数学直觉、实现解释和简单数值例子。
- `docs/RMSNorm.md`
  介绍 RMSNorm 的公式、数学直觉、实现解释和简单数值例子。
- `docs/summary.md`
  对三种归一化方法做整体总结，重点比较核心思想、统计方式和适用场景。

## 这一部分主要讲什么

Normalization 的核心目标，是让网络中的激活值保持在更稳定的数值范围内，从而改善训练稳定性和优化效率。

虽然三种方法都在做“尺度控制”，但它们的统计方式不同：

- `BatchNorm`：在 batch 内跨样本统计同一特征维；
- `LayerNorm`：在单个样本内部统计最后一维；
- `RMSNorm`：不减均值，只控制单个样本向量的整体幅值。

也正因为统计方式不同，它们适用的模型类型也不同。

## 核心结论

- `BatchNorm` 更适合 CNN 一类模型。
  因为图像任务中 batch 内样本分布通常更稳定，而且卷积网络常能使用更大的 batch size，使得 batch 统计量更可靠。
- `LayerNorm` 更适合 Transformer。
  因为它不依赖 batch，能稳定处理变长序列和较小 batch size。
- `RMSNorm` 是 LayerNorm 的更简洁变体。
  它不做减均值，只控制整体尺度，因此在很多现代大模型中更常见。

如果从 LLM 的角度看：

- `BatchNorm` 往往不是首选，因为语言数据在不同 batch 之间波动较大，batch 统计不稳定；
- `LayerNorm` 和 `RMSNorm` 更符合 token 级表示学习的需求。

## 代码实现说明

本目录下的代码实现都偏向“最小可解释版本”，特点是：

- 公式直接对应代码，便于学习；
- 保留了可学习参数；
- 增加了基本的形状检查；
- 更强调原理清晰，而不是完整工程封装。

因此，这里的实现适合：

- 学习归一化原理；
- 对照公式理解代码；
- 作为后续扩展到完整模型实现的基础版本。

但也需要注意：

- `BatchNorm` 当前不是完整工业版，没有维护推理阶段常用的运行统计量；
- 这些实现更偏教学，不是直接面向大规模训练框架的最终模块。

## 一句话总结

这一部分可以概括为：

`BatchNorm` 依赖 batch 统计，适合 CNN；`LayerNorm` 依赖样本内部统计，适合 Transformer；`RMSNorm` 只控制向量能量，适合现代大模型中的轻量归一化。
